# MyStonks API 测试使用指南

## 快速开始

### 1. 安装依赖

```bash
cd D:\project\my_stonks_api_tests
go mod tidy
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，填写以下配置：

```env
# API基础配置
API_BASE_URL=http://localhost:8080

# 管理端 Token（用于 /api/v1/admin/* 接口）
ADMIN_TOKEN=your_admin_token_here

# 用户端 Token（用于 /api/v1/wth/* 等用户端接口）
USER_TOKEN=your_user_token_here

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=your_password
DB_NAME=my_stonks

# 测试配置
TEST_TIMEOUT=30
```

**Token类型说明：**
- `ADMIN_TOKEN`：用于管理端接口（路径包含 `/admin/`）
  - 例如：`/api/v1/admin/wth/coin/add`
  - 例如：`/api/v1/admin/wth/product/add`
- `USER_TOKEN`：用于用户端接口（路径不包含 `/admin/`）
  - 例如：`/api/v1/wth/coin/selectCoin`
  - 例如：`/api/v1/wth/user/order/create`

**自动Token选择：**
测试框架会根据接口路径自动选择正确的Token类型：
- 如果路径包含 `/admin/` → 使用 `ADMIN_TOKEN`
- 如果路径不包含 `/admin/` → 使用 `USER_TOKEN`

### 3. 运行测试

```bash
# 运行所有测试
make test

# 运行测试（详细模式）
make test-verbose

# 生成覆盖率报告
make test-coverage

# 清理测试报告
make clean
```

## 测试用例编写规范

### 基本结构

所有测试用例遵循以下结构：

```go
package cases

import (
    "testing"
    "my_stonks_api_tests/config"
    "my_stonks_api_tests/framework"
    "my_stonks_api_tests/models"
    "my_stonks_api_tests/testdata"
    _ "my_stonks_api_tests/config"
)

func TestModuleName_Action(t *testing.T) {
    // 1. 初始化客户端和数据库
    client := framework.NewTestClient()
    db := framework.NewDBClient(t)
    defer db.Cleanup()

    // 2. 表格驱动测试用例
    tests := []struct {
        name           string
        request        interface{}
        setup          func()
        verify         func()
        wantSuccess    bool
        wantMsgContain string
    }{
        {
            name:        "正常场景",
            request:     testdata.NewTestData(),
            wantSuccess: true,
            verify: func() {
                // 通过数据库验证
                var record models.YourModel
                err := db.GetByCondition(&record, "condition = ?", "value")
                framework.AssertDBRecordExists(t, err == nil, err)
                framework.AssertDBFieldEqual(t, "Field", expected, actual)

                // 清理测试数据
                db.DeleteTestData(&models.YourModel{}, "condition = ?", "value")
            },
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            // Setup - 准备
            if tt.setup != nil {
                tt.setup()
            }

            // Act - 执行
            resp, err := client.Post("/api/endpoint", tt.request)

            // Assert - 验证
            if tt.wantSuccess {
                if err != nil {
                    t.Errorf("请求失败: %v", err)
                    return
                }
                framework.AssertStatusCode(t, resp, 200)

                // 通过数据库验证
                if tt.verify != nil {
                    tt.verify()
                }
            } else {
                if err != nil {
                    t.Errorf("请求失败: %v", err)
                    return
                }
                framework.AssertStatusCode(t, resp, 200)
                if tt.wantMsgContain != "" {
                    framework.AssertErrorMessageContains(t, resp, tt.wantMsgContain)
                }
            }
        })
    }
}
```

### 三段式结构

每个测试用例内部应该遵循三段式结构：

#### 1. Arrange（准备）
```go
framework.LogTestStep(t, 1, "准备测试环境")

// 准备前置条件
if tt.setup != nil {
    tt.setup()
}
```

#### 2. Act（执行）
```go
framework.LogTestStep(t, 2, "调用API接口")

// 调用API
resp, err := client.Post("/api/endpoint", tt.request)
```

#### 3. Assert（验证）
```go
framework.LogTestStep(t, 3, "验证测试结果")

// 通过数据库验证
if tt.verify != nil {
    tt.verify()
}

// 断言
framework.AssertStatusCode(t, resp, 200)
framework.AssertDBFieldEqual(t, "Field", expected, actual)
```

## 数据库验证

### 查询单条记录
```go
var coin models.WthCoinConfig
err := db.GetByCondition(&coin, "coin_key = ?", "BTC")
framework.AssertDBRecordExists(t, err == nil, err)
framework.AssertDBFieldEqual(t, "CoinKey", "BTC", coin.CoinKey)
```

### 查询记录数量
```go
count, err := db.GetCount(&models.WthCoinConfig{}, "shelves = ?", 1)
framework.AssertDBCount(t, expected, count, err)
```

### 检查记录是否存在
```go
exists, err := db.Exists(&models.WthCoinConfig{}, "coin_key = ?", "BTC")
framework.AssertDBRecordExists(t, exists, err)
```

### 删除测试数据
```go
db.DeleteTestData(&models.WthCoinConfig{}, "coin_key = ?", "TEST001")
```

## 断言辅助函数

### HTTP相关
- `AssertStatusCode(t, resp, expectedCode)` - 断言HTTP状态码
- `AssertSuccess(t, resp)` - 断言API返回成功
- `AssertAPIError(t, resp, expectedCode)` - 断言API返回错误
- `AssertErrorMessageContains(t, resp, msg)` - 断言错误消息包含指定内容

### 数据库相关
- `AssertDBRecordExists(t, exists, err)` - 断言数据库记录存在
- `AssertDBRecordNotExists(t, exists, err)` - 断言数据库记录不存在
- `AssertDBCount(t, expected, actual, err)` - 断言记录数量
- `AssertDBFieldEqual(t, field, expected, actual)` - 断言字段值相等
- `AssertDBFieldNotEqual(t, field, notExpected, actual)` - 断言字段值不相等

### 日志相关
- `LogTestStep(t, step, description)` - 打印测试步骤
- `LogTestResult(t, success, message)` - 打印测试结果
- `LogDBQuery(t, table, condition, result)` - 打印数据库查询结果

## 注意事项

1. **只读数据库**：数据库只做查询验证，不做任何增删改操作
2. **数据准备**：测试数据通过API接口创建，而不是直接插入数据库
3. **数据验证**：所有数据验证通过数据库查询完成，确保API正确执行
4. **独立性**：每个测试用例应该独立运行，不依赖其他用例
5. **清晰的日志**：使用LogTestStep和LogTestResult输出清晰的测试日志
6. **错误处理**：所有错误都应该被捕获和处理，避免测试直接panic
7. **Token配置**：确保在`.env`文件中正确配置了`ADMIN_TOKEN`和`USER_TOKEN`
8. **Token类型**：测试框架会根据接口路径自动选择对应的Token类型，无需手动指定
